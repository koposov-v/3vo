Разработать микросервисный API, базирующийся на 3-х микросервисах.
1. API Gateway - REST API для клиентов (мобильное приложение, веб-сайт)
- разработать эндпоинты для создания, получения, обновления и отмены заказов
- аутентификация и авторизация пользователей
- взаимодействие со вторым сервисом по gRPC

2. Сервис заказов (gRPC) - обработка и хранение заказов
- gRPC API для взаимодействия с API Gateway
- взаимодействие с сервисом кухни по gRPC
- хранение данных

3. Сервис кухни (gRPC) - управление очередью заказов для приготовления
- gRPC API для взаимодействия с сервисом заказов
- отслеживание статусов приготовления
- уведомление сервиса заказов об изменении статуса

Хранение данных на усмотрение разработчика: можно просто держать в памяти или хранить в PostgreSQL.
Все микросервисы должны быть упакованы в Docker контейнеры.
Необходимо настроить их совместный запуск с помощью Docker Compose.
